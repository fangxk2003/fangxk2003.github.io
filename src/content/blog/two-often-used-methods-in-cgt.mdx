## Grundy Number and Mex

[Combinatorial Game Theory | Set 3 (Grundy Numbers/Numbers and Mex)](https://www.geeksforgeeks.org/dsa/combinatorial-game-theory-set-3-grundy-numbers-numbers-and-mex/)

This is a useful introductory article on Grundy numbers.

### Grundy Number

A [Grundy number](https://en.wikipedia.org/wiki/Nimber) is a value assigned to a game state. For many [impartial games](https://www.geeksforgeeks.org/dsa/introduction-to-combinatorial-game-theory/), such as Nim, the Grundy number captures the strategic meaning of the position.

The key idea is that every position behaves like a Nim pile of a certain size. A position with Grundy number $0$ is losing for the next player, while a position with a nonzero Grundy number is winning if both players play optimally.

### Minimum Excludant (Mex)

The minimum excludant, usually written as "mex", of a set of numbers is the smallest non-negative integer not present in that set.

![MeX](/images/two-often-used-methods-in-cgt/MeX.png)

$$
G(S)=\operatorname{mex}\{G(T): T \text{ is reachable from } S \text{ in one move}\}
$$

This recurrence is especially useful in Nim-like games: compute the Grundy numbers of all positions reachable in one move, then choose the smallest non-negative integer that does not appear among them.

Two Project Euler problems where this technique appears:

[PE-P306](https://projecteuler.net/problem=306)

[PE-P310](https://projecteuler.net/problem=310)

## Alpha-Beta Pruning

Technically, alpha-beta pruning is not a CGT method in the same sense as Grundy numbers. It is a search optimization for minimax trees, but it often appears in game-solving problems.

[Alpha-Beta Pruning Wiki](https://en.wikipedia.org/wiki/Alpha%E2%80%93beta_pruning)

![Alpha-beta pruning](/images/two-often-used-methods-in-cgt/AB_pruning.png)

Alpha-beta pruning stops evaluating a move once it is clear that the move cannot improve the final decision. In other words, if one branch has already proved that a position is worse than a previously examined choice, the remaining branches under that position do not need to be explored.

When applied to a standard minimax tree, alpha-beta pruning returns the same move as ordinary minimax. The difference is that it skips branches that cannot possibly affect the result, which can reduce the search space dramatically.

It is used in [PE-P505](https://projecteuler.net/problem=505).
